# PLAYBACK Engine — Improvement Plan

**Created:** 2026-06-26
**Owner:** Karim + Claude
**Companion docs:** `playback_engine_modules.md` (module map), `portrait-crop-v5-analysis.md` (failure analysis), `scripts/portrait-crop/PORTRAIT-CROP.md` (pipeline).

This is the master checklist. We tackle it phase by phase, top to bottom. Every code change is **eval-gated** (`scripts/portrait-crop/eval.ts` against the pinned baseline) so we can prove improvement and never silently regress.

---

## Guiding principles

1. **Eval-gated, always.** No change ships without running `eval.ts` on the 15-clip dataset and comparing to the pinned `results/<sha>.json`. Quality is a number, not a vibe.
2. **Rank by downstream-product-unlock.** A BallDetector/PlayerTracker fix helps PortraitCrop _and_ PerPlayerHighlights _and_ AutoFollow → higher priority than a single-product fix.
3. **Don't rebuild what Veo gives free.** Veo content already has AI goal/shot tags (cached in `playhub_recording_events`). Spend ML effort where we're differentiated: Spiideo/direct uploads + per-player.
4. **Highlights are a ranking problem, not a binary-classification problem** (the Taka learning — see Phase 4). This sidesteps the ~57% goal-detection ceiling.
5. **Modular.** Each module ships and is pinned independently.

**Already done (do NOT redo):** Forzasys soccer YOLO weights, Norfair Kalman tracking, IQR outlier filter, Savitzky-Golay smoothing, segmentation, speed-clamping, conditional SAHI, ffmpeg `sendcmd` per-frame render, eval harness + sweepers.

## Phase 2 — Harden the BallDetector + add FieldGeometry (the shared primitives)

> 🔴 **DATA-CONFIRMED TOP PRIORITY (2026-06-27).** First GT-backed eval on the hero clip (`veo_20260502_goal_01`, dense frames 29–523) gives **`recall@p0.9` = 16%** and **GT-backed ball-in-crop = 69%** (vs the self-referential 87% that flattered itself). The detector finds the ball on only ~16% of frames where it's actually visible → the **BallDetector is the dominant limiter**, not the crop logic. Caveat: one clip, the hardest (goal = fast/blurred ball); need a few more labeled clips to firm it up, but the signal is unambiguous. This re-orders the roadmap with data: Phase 2 is now ahead of further Phase 1 crop polish.

_Why: BallDetector quality is the ceiling for PortraitCrop AND every future product. FieldGeometry unlocks edge-rejection now and off-ball metrics later._

> **Failure mode discovered 2026-06-27 (task 8, passage):** on grassroots, a SECOND match often plays on an adjacent pitch in-frame. The detector accurately locked onto the _neighbouring_ match's ball (far from our action) instead of the main ball. This is a **semantic selection** problem, not detection — a pure constant-velocity Viterbi DP can lock onto the wrong-match ball (it may have a _cleaner_ trajectory than our occluded one). **Implication:** the DP trajectory cost must include a **spatial/region prior** (player-cluster centroid as a soft term, and/or a field-of-play homography polygon) in addition to motion. This **elevates FieldGeometry (2.3)** from nice-to-have to part of the selection layer. NOTE: the Step-1 deletion of the crude cluster-proximity _boost_ may make adjacent-pitch clips look worse — that's expected; the proper fix is a _soft_ region term in the DP, which keeps set-pieces AND rejects the neighbour's ball.

> **Related failure mode 2026-06-27 (task 5, NCFE practical):** some clips have MULTIPLE balls _inside_ our field (training drills). The single-ball assumption breaks here too. Architecture answer is the same: detect ALL balls (multi-hypothesis), let the relevance prior (player-proximity / dominant-trajectory) pick _the_ ball to follow. **Product framing:** matches = one ball (core); multi-ball _drills_ are an edge case with no well-defined target. **Labeling policy:** on multi-ball clips, label the _active match ball_ consistently as the target — OR exclude pure drill clips from the eval set (the "right answer" is ambiguous and can mislead the metric). The NCFE clip is a coaching practical, not a match → candidate for exclusion; prefer real match clips for the holdout.

> **DECISION 2026-06-27 — Phase C (detector) BEFORE Plan B; the airborne ball is a MOTION problem (ml-ops + CV + prior-art, 3-way convergent).** Holdout validation showed grassroots GOALS are detector-limited: cup-final goal **oracle (raw-candidate recall) = 10%** (in-crop 0%). Plan A AND Plan B are both SELECTION layers capped by the oracle → neither can beat 10% on goals → fix DETECTION first. **KEY INSIGHT:** the airborne ball (white-on-white, motion-blurred) has ~no static texture → an appearance fine-tune is the WRONG/last lever; **MOTION is the cue**, and our DP already consumes a candidate set → motion proposals integrate as extra candidates the DP selects (don't need confidence, just presence). **Cheap path (no labels/GPU), in order:** (0) BUG FIX — `SKY_CUTOFF_FRAC` was 0.15, deleting airborne candidates in the top 15% of frame (where the goal ball flies) BEFORE the DP; set to 0, re-measure oracle [running 2026-06-27]. (1) motion proposals (frame-diff/bg-sub → small moving blobs → candidates conf≈0.25, `source=motion`) injected into the candidate list `select_trajectory` consumes [~1-2 days]. (2) if still short: TOTNet/TrackNet temporal heatmap ([github.com/AugustRushG/TOTNet](https://github.com/AugustRushG/TOTNet)) or a PROPER fine-tune. **FAILED-FINETUNE AUTOPSY:** trained from generic `yolov8m.pt` NOT `yolov8m_forzasys_soccer.pt` (lost the broadcast ball prior) + catastrophic forgetting (freeze=10 / 50ep / lr1e-3 / 330 easy imgs). **EVAL-GATE BUG:** manifest `frozen_holdout` is empty `[]` — populate before any retrain. **Proper fine-tune (only if motion insufficient):** from Forzasys, head-only freeze, lr≤5e-4, ≤20ep early-stop, blur+scale aug, composition 40% airborne / 25% normal / 20% hard-neg / 15% broadcast-replay, temporal split by match, gate = beat current pipeline on oracle AND in-crop with no per-clip regression. **Plan B deferred** until oracle on goals clears ~40%.

> **Motion-proposal experiment RESULT 2026-06-27 — cheap-path CEILING found.** Built double-difference motion candidates → injected into the DP candidate set. **VALIDATED THE THESIS:** airborne-ball oracle **10%→100%** (motion sees the ball perfectly; confirmed it IS recoverable). **BUT net-negative as integrated:** (a) cup-final in-crop stayed **0%** — the DP picks a slow distractor (x≈1500, stationary) over the fast airborne ball (x≈600, descending from y≈78), on 50/50 frames, because raw motion gives EVERY moving blob the same conf=0.25, so the DP tie-breaks on trajectory consistency and a _stationary distractor beats a fast ball_; (b) regressed the hero goal **86→61** (≈90 noise candidates/frame drowned the good YOLO detections). **Raw motion = high-recall / ZERO-precision.** DISABLED (`MOTION_PROPOSALS=False`); code kept for future gated use. **CONCLUSION: the cheap path is exhausted for airborne goals.** The real fix is a **LEARNED TEMPORAL DETECTOR** (TOTNet [github.com/AugustRushG/TOTNet](https://github.com/AugustRushG/TOTNet) / TrackNetV2/3) that outputs a CLEAN, high-confidence ball heatmap (few candidates, fires on the ball not distractors) — prior-art #1. That is the proper Phase C: ~few-hundred labeled airborne frames + a Modal training run + populate the empty `frozen_holdout`. **LOCKED WINS that stand (motion off):** Plan-A motion-DP — goal in-crop 62→86%, goalkick 100%, no regressions.

> **Phase C MVP — TrackNetV2 temporal detector (VALIDATED 2026-06-27 by CV + ml-ops + prior-art; scaffolding started).** Model = **TrackNetV2** fine-tuned from badminton-pretrained weights ([ChgygLin/TrackNetV2-pytorch](https://github.com/ChgygLin/TrackNetV2-pytorch); TOTNet is a v2 upgrade, too heavy for the gate). **Two corrections to the naive recipe (CV specialist):** (1) do NOT global-downscale 1920→512 (a 12px ball → 3px, dead) — band-crop the playable strip + run ~1280×704 or tile it; (2) **5-frame, CENTER-predicted**, not 3-frame causal (our ball is too fast; offline so lookahead is free). **Why it fixes the DP:** heatmap peak height = real calibrated conf → `-log(conf)` finally separates ball (~0.9) from distractors (~0.2) — the signal raw motion lacked. Drop-in at the disabled motion hook, ZERO DP change. **Complement, not replace YOLO** (YOLO=ground ball, TrackNet=airborne, DP fuses; per-source conf calibration needed). **DONE:** committed `eval-dataset/oracle.mjs` (per-source gate metric; reproduces yolo=10%/motion=100% on cup-final) + `frozen_holdout=["veo_20260506_hb_cupfinal_goal_01"]`. **GATE (ml-ops):** on the frozen-holdout airborne frames, oracle ≥60% (apex ≥40%) = PROCEED to full labeling; <40% = ABANDON (baseline 10%). Also log true-ball candidate RANK (avoid re-creating motion's zero-precision trap). Measure YOLO + TrackNet with the SAME oracle.mjs, same frames, same run. **TODO scaffolding:** clone TrackNet repo, data-prep (CVAT bbox→Gaussian heatmap, 5-frame stacks, band-crop), Modal training app, heatmap→candidate adapter (`source:"temporal"`, conf=peak). **USER labeling (critical path):** dense-label airborne WINDOWS (25fps) of 2-3 goal clips + the holdout, phase-tagged (ascending/apex/descending); ~80-150 frames, ~3-4h. Split BY MATCH (no frame leakage).
> **Phase C MVP — SCAFFOLDING BUILT 2026-06-27** (in `scripts/portrait-crop/tracknet/`): `vendor/` = cloned ChgygLin/TrackNetV2-pytorch (3-in-3-out, on-the-fly Gaussian heatmaps); `prep_dataset.py` = CVAT labels+videos → TrackNet dataset (frames + `frame_num,visible,x,y`-normalized CSV + `match.yaml`, val=frozen_holdout) — VERIFIED (1139 frames from the 5 labeled clips); `detect_tracknet.py` = inference adapter, heatmap top-K peaks → our `_raw.json all_candidates` (`source:"temporal"`, conf=peak height) so `oracle.mjs` reads it apples-to-apples; `train_modal.py` = Modal A10G fine-tune (train HIGHER than default 288×512 — use 720×1280, no global downscale). **Run sequence:** (1) user labels 2-3 airborne-goal clips → `python prep_dataset.py`; (2) `modal run train_modal.py --imgsz-h 720 --imgsz-w 1280`; (3) `python detect_tracknet.py --source <holdout>.mp4 --weights best.pt --imgsz 720 1280 --out <holdout>_raw.json`; (4) `node eval-dataset/oracle.mjs <holdout>` → temporal oracle ≥60% (apex ≥40%) vs YOLO 10% = PROCEED.

> **✅ FINAL / SHIPPED 2026-06-29 — supersedes the action-region pivot AND the TrackNet plan for now.** The product owner validated the actual videos and the answer is simpler than all of the above: **ball-following crop + full-height (no zoom) + a human-correction override.** Key corrections to earlier conclusions: (1) the "action-region / drop ball-tracking" pivot was **REJECTED** — the owner wants the BALL in frame, not just the action (a goal clip where you "barely see the ball" is terrible to the actual user, regardless of the UX-expert's generic advice). (2) The zoom I added was **unnecessary** — it was solving an off-ball symptom; removed (now `CROP_ZOOM` env, default 1.0 = full height). (3) **TrackNet is NOT needed**: the airborne/clearance detector blind spots are covered by a few-minutes CVAT hand-correction of the ball track, fed via `CROP_BALL_LABELS=<labels.json>` which overrides the detector inside the labeled window. **Shipped crop.ts:** follow the ball (+Kalman `tracked`), HOLD last position when lost (no center-bias, no drift-to-cluster); `CROP_BALL_LABELS` human override; `CROP_ZOOM` tunable. **Validated:** goalkick = PERFECT end-to-end (owner-confirmed) once the owner corrected its full ball track in CVAT; the detector alone is "perfect" on ground play and only fails on fast airborne/long balls — which the override fixes per-clip. **Workflow:** auto-crop everything → for clips needing polish (airborne goals/clearances), correct the ball in CVAT on the hard frames → re-crop with the override. **Gotcha:** label the window that MATTERS (the goal moment), not the buildup — the cup-final first attempt labeled t=0–1.7s while the goal was at t≈9s, so the goal stayed detector-driven/lost. TrackNet scaffolding + Plan B (field geometry) remain on the shelf as the full-auto path if hand-correction ever becomes too costly at volume.

> **Plan B — Field-of-Play geometry (VALIDATED 2026-06-27 by CV specialist + prior-art; build AFTER detector oracle on goals clears ~40%).** The soft player-cluster _centroid_ prior failed because it's a POINT prior — on a freekick the centroid sits between the real ball (x≈870) and the FP (x≈1500), and a goal-shot is equidistant (~500px) to the FP from the cluster, so distance-to-players can't separate them. Fix = replace the POINT prior with a SHAPE prior (the field of play). Validated approach, ranked:
>
> 1. **Static per-venue ROI polygon = PRIMARY.** Cameras are FIXED per venue → annotate the pitch polygon once per camera install (what Veo/Spiideo/Pixellot do). Hard-kills adjacent-pitch + distant FPs; most robust, ~zero compute, no per-clip estimation risk. Persist on the venue (schema change → senior-code-reviewer + database-performance-optimizer).
> 2. **Accumulated player-occupancy map = zero-effort AUTO-BOOTSTRAP for the polygon + per-clip fallback.** Accumulate player FOOT-points (box x-extent at y2, NOT just centroid) over the clip → percentile threshold → morphological close → largest connected component → dilate outward (protect goal-mouth) → polygon. Adjacent-pitch = disjoint blob (near-certain fix). Two-pitch tie-break: keep the CC overlapping the tracked-ball density; if ambiguous → disable.
> 3. **Integrate as a SOFT node cost** (reuse the `DP_REGION_*` slot: `c += DP_REGION_WEIGHT*(1-occ)`), NOT a hard filter — graceful degradation, lets the DP arbitrate boundary cases. Region-validity gate → degrade to bit-identical motion-DP if player coverage is sparse (U9) or ambiguous.
> 4. **Ball-size-vs-y prior** = cheap orthogonal add-on (apparent ball size monotonic in y on a fixed camera; catches wrong-depth FPs).
> 5. **SKIP homography** (TVCalib/PnLCalib) — needs broadcast pitch lines; fails on grassroots/panoramic.
>    **CRITICAL pre-build test (standing rule):** "occupancy separates them" is a hypothesis until measured — get occ(870)/occ(1500) on the freekick clip FIRST (need ≥~2-3×). Adjacent-pitch is the near-certain win; the freekick fix hinges on that number.

> **Failure-mode catalogue (observed during labeling 2026-06-27, map to the two fixes):** (1) fast shot → IQR rejected it [Step-2 DP]; (2) adjacent-pitch ball [Step-2 region prior]; (3) multi-ball drill [Step-2 + labeling policy]; (4) **airborne ball missed** (blur/small vs sky) → genuine detector gap [Phase-C fine-tune: airborne/blur aug + higher-res; DP bridges short gaps via ballistic motion + occlusion node]; (5) **white shoes detected as ball** = FP noise [Step-2 DP rejects via motion/region; + white-object hard negatives for Phase-C]. → Fine-tune corpus spec emerging: mine airborne-ball TPs + white-object hard negatives.

## Phase 4 — Highlights as ranking + structured event index (the Taka learning)

_Why: reframes the killed binary goal-detector as a graceful ranker, and the event index powers stats, search, dev programs, and the recruiting graph (ties to the 3STEP/US-recruiting opportunity)._

- [ ] **4.1 Structured event index — NEW.** Every detected action → `{match, player_id, action_type, zone, t_start, t_end, quality_score, on_off_ball}`. First-class Engine output (not just rendered clips). Persist to Supabase.
- [ ] **4.2 ActionClassifier + quality scoring.** Score the _quality/salience_ of each action (ball-proximity, pose, motion, zone) and surface top-N. Ranking degrades gracefully where binary classification fails.
- [ ] **4.3 Off-ball metrics.** From PlayerTracker tracklets + FieldGeometry homography: runs into space, pressing, defensive line, positioning — the recruiting/development differentiator Veo lacks.
- [ ] **4.4 "Assistant Coach" NL retrieval.** Thin LLM/RAG layer over the event index ("show my #9's shots in the box," "opponent set-piece routines"). Cheap once 4.1 exists; reuses the existing AI layer/API.
- [ ] **4.5 EventSpotter v5 for non-Veo content only.** SoccerNet-pretrained ResNet-152 + temporal transformer (per competitive research) for Spiideo/direct uploads where Veo tags are absent. Gated; do not rebuild for Veo content.

**Exit criteria:** event index populated for a match; top-N ranked highlights beat the current goal-only output on a human-rated sample; one working NL query.
